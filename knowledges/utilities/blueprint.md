# Utility Guide: CLI Blueprint (`blueprint`)

Blueprint is a powerful DSL-based template system in Skalfa API. It allows you to define your database schema, relations, seeders, and controllers in a single JSON file. The Blueprint engine then automatically generates the corresponding Models, Migrations, Controllers, and Seeders.

---

## 1. Defining a Blueprint File

Blueprint files are JSON files located in the `blueprints/` directory at the project root and must end with `.blueprint.json` (e.g., `blueprints/product.blueprint.json`).

Each file must contain an array of blueprint schemas:

```json
[
  {
    "model": "catalog/product",
    "schema": {
      "name":        "type:string,200 required min:1 max:200 fillable searchable selectable",
      "sku":         "type:string,50 unique index required fillable searchable selectable",
      "price":       "type:float required min:0 fillable selectable",
      "image":       "type:string,255 nullable fillable selectable file",
      "description": "type:string nullable fillable selectable"
    },
    "relations": {
      "category": "catalog/category fillable",
      "tags":     "[]:catalog/tag expandable fillable"
    },
    "controllers": [
      "catalog/product/product catalog:products 300:produk"
    ],
    "seeders": [
      ["Smartphone X", "SM-X100", 999.99, "/photos/sm-x100.jpg", "Flagship phone"],
      ["Laptop Pro", "LP-P200", 1499.00, null, "Professional laptop"]
    ]
  }
]
```

---

## 2. Schema Definition DSL

Inside the `"schema"` block, keys represent column names. The value is a space-separated string of DSL options:

### A. Data Types (`type:<type>[,length]`)
*   `type:string[,length]` (e.g. `type:string,100` - defaults to 255 if length is omitted).
*   `type:int`
*   `type:float`
*   `type:boolean`
*   `type:date`
*   `type:timestamp`

### B. Database Constraints & Indexing
*   `required`: Column is NOT NULL.
*   `nullable`: Column can be NULL.
*   `unique`: Adds a UNIQUE constraint.
*   `index`: Adds a database INDEX.

### C. ORM Behavior Flags
*   `fillable`: Allows the field to be populated via `.pump()`.
*   `selectable`: Includes the field in default SELECT queries.
*   `searchable`: Includes the field in global `.search()` queries.
*   `hidden`: Hides the field from default JSON serialization (e.g., `password`).
*   `file`: Marks the field as a file upload path.

### D. Validation Rules
*   `min:N`: Minimum length (strings) or value (numbers).
*   `max:N`: Maximum length (strings) or value (numbers).

---

## 3. Relations Definition

Inside the `"relations"` block, the value defines the relationship type and target model:

*   `TargetModel`: **BelongsTo** relation (no prefix).
*   `[]:TargetModel`: **HasMany** or **BelongsToMany** relation.
*   `[1]TargetModel`: **HasOne** relation.
*   *Flags*: Add `fillable` or `expandable` to configure ORM behavior.

---

## 4. Controllers Definition

The `"controllers"` block defines the API endpoints. It is an array of strings in the following format:
`"path-controller namespace-route:route-path key-access:label-access"`

Example:
`"catalog/product/product catalog:products 300:produk"`
*   **Controller Path**: `app/controllers/catalog/product/product.controller.ts`
*   **Route Prefix**: `/api/catalog/products`
*   **Permission Key**: `300` (e.g. `300.01` for View, `300.02` for Create, etc.).

*Note: Set `"controllers": false` if you do not want to generate a controller.*

---

## 5. Running the Blueprint Engine

To generate all models, migrations, and controllers, run the following CLI command:

```bash
bun skalfa blueprint
```

### Running Specific Blueprints:
You can run only specific blueprints using the `--only` flag:
```bash
bun skalfa blueprint --only product
```

---

## 6. Customizing Generated Files

Every generated file contains a special header comment:
```typescript
// ============================================
// ## file THIS FILE IS AUTO-GENERATED BY BLUEPRINT
// ?? Blueprint : product.blueprint.json
// !! If this comment is removed, blueprint engine WILL NOT override this file.
// ============================================
```

If you need to write custom logic in a generated controller or model, **remove this header comment**. Once removed, the Blueprint engine will **never** overwrite or modify this file when re-running the generator.
